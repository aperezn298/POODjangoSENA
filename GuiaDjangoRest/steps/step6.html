En este paso aprenderemos a probar nuestra API REST utilizando Postman, una herramienta esencial para el desarrollo y pruebas de APIs.

Pruebas con Postman

Postman es una plataforma para desarrollo de API que permite diseñar, probar, documentar y compartir APIs. Es una herramienta indispensable para cualquier desarrollador que trabaje con APIs.

Beneficios de Postman

Facilidad de Uso

Interfaz intuitiva que permite realizar solicitudes HTTP sin necesidad de escribir código.

Ideal para: Desarrollo rápido y pruebas

Características: Historial de solicitudes, entornos configurables

Colecciones y Flujos

Organiza solicitudes en colecciones y ejecuta flujos completos para simular casos de uso reales.

Ideal para: Pruebas de integración

Características: Ejecución secuencial, variables dinámicas

Automatización

Crea pruebas automatizadas con scripts pre-request y post-request usando JavaScript.

Ideal para: CI/CD y pruebas repetitivas

Características: Assertions, reportes de pruebas

Paso 6.1: Instalación y Configuración de Postman

Para comenzar a probar nuestra API, primero debemos instalar y configurar Postman:

1. Descargar e Instalar Postman

Visita postman.com/downloads y descarga la versión correspondiente a tu sistema operativo. Sigue las instrucciones de instalación.

Alternativa: También puedes usar Postman directamente en tu navegador desde go.postman.co/build si no deseas instalar la aplicación de escritorio.

2. Crear una colección

Las colecciones son grupos de solicitudes relacionadas. Vamos a crear una colección para nuestra API:

1. Abre Postman y haz clic en "Collections" en el panel izquierdo
2. Haz clic en el botón "+" para crear una nueva colección
3. Nombra la colección "Django REST API"
4. Puedes añadir una descripción opcional
5. Haz clic en "Create"

3. Configurar un entorno

Los entornos en Postman permiten definir variables reutilizables para diferentes contextos (desarrollo, producción, etc.):

1. Haz clic en "Environments" en el panel izquierdo
2. Haz clic en "+" para crear un nuevo entorno
3. Nombra el entorno "Development"
4. Añade estas variables:
   - base_url: http://127.0.0.1:8000
   - api_version: api
   - token: (déjalo en blanco por ahora)
5. Haz clic en "Save"
Consejo: Para usar estas variables, utiliza la sintaxis {{variable}} en tus solicitudes, por ejemplo: {{base_url}}/{{api_version}}/products/

4. Configurar las cabeceras globales

Para trabajar con APIs REST, necesitamos configurar algunas cabeceras HTTP comunes:

{
    "Content-Type": "application/json",
    "Accept": "application/json"
}

Paso 6.2: Probando Endpoints sin Autenticación

Vamos a empezar probando los endpoints que no requieren autenticación (si los hay). Asegurémonos de que nuestra API está funcionando:

1. Verificar que el servidor Django esté corriendo

# Iniciar el servidor Django si no está corriendo
python manage.py runserver

2. Probar un endpoint de lectura (GET)

Vamos a crear nuestra primera solicitud GET en Postman:

1. En tu colección "Django REST API", haz clic en "..." > "Add Request"
2. Nombra la solicitud "Get All Products" (o el recurso correspondiente a tu API)
3. Establece el método en GET
4. En el campo URL, escribe: {{base_url}}/{{api_version}}/products/
5. Asegúrate de que el entorno "Development" esté seleccionado
6. Haz clic en "Send"

Si todo está configurado correctamente, deberías recibir una respuesta con un código de estado 200 y datos en formato JSON.

Consejo: Guarda las solicitudes útiles haciendo clic en "Save". Esto te permitirá reutilizarlas en el futuro sin tener que configurarlas nuevamente.

Paso 6.3: Probando la Autenticación

Para probar endpoints protegidos, primero debemos obtener un token de autenticación:

1. Obtener un token de autenticación

1. Crea una nueva solicitud en tu colección
2. Nombra la solicitud "Get Auth Token"
3. Establece el método en POST
4. En el campo URL, escribe: {{base_url}}/api-token-auth/
5. Ve a la pestaña "Body"
6. Selecciona "raw" y "JSON" en el menú desplegable
7. Ingresa las credenciales de un usuario existente:
   {
     "username": "tu_usuario",
     "password": "tu_contraseña"
   }
8. Haz clic en "Send"

La respuesta debería incluir un token de autenticación:

{
    "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b"
}

2. Guardar el token como variable de entorno

1. Haz clic derecho en el valor del token en la respuesta
2. Selecciona "Set as variable" > "Development" > "token"
3. El token ahora está guardado y puede usarse en otras solicitudes

3. Configurar Gunicorn como servicio

Crea un archivo de servicio systemd:

sudo nano /etc/systemd/system/gunicorn_mi_api.service

Añade el siguiente contenido:

[Unit]
Description=gunicorn daemon for mi_api_django
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/mi_api_django
ExecStart=/var/www/mi_api_django/venv/bin/gunicorn --access-logfile - --workers 3 --bind unix:/var/www/mi_api_django/mi_api_django.sock mi_proyecto.wsgi:application
Restart=on-failure

[Install]
WantedBy=multi-user.target

Inicia el servicio:

sudo systemctl start gunicorn_mi_api
sudo systemctl enable gunicorn_mi_api

4. Configurar Nginx

Crea una configuración para Nginx:

sudo nano /etc/nginx/sites-available/mi_api_django

Añade el siguiente contenido:

server {
    listen 80;
    server_name tudominio.com www.tudominio.com;

    location = /favicon.ico { access_log off; log_not_found off; }
    
    location /static/ {
        root /var/www/mi_api_django;
    }

    location /media/ {
        root /var/www/mi_api_django;
    }

    location / {
        include proxy_params;
        proxy_pass http://unix:/var/www/mi_api_django/mi_api_django.sock;
    }
}

Habilita el sitio y reinicia Nginx:

sudo ln -s /etc/nginx/sites-available/mi_api_django /etc/nginx/sites-enabled
sudo nginx -t  # Verificar configuración
sudo systemctl restart nginx

5. Configurar SSL con Let's Encrypt

# Instalar Certbot
sudo apt install -y certbot python3-certbot-nginx

# Obtener certificado SSL
sudo certbot --nginx -d tudominio.com -d www.tudominio.com

Paso 6.4: Despliegue con Docker (alternativa)

Si prefieres usar Docker, aquí hay una configuración básica:

1. Crear Dockerfile

FROM python:3.10-slim

ENV PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1 \
    PIP_NO_CACHE_DIR=off \
    PIP_DISABLE_PIP_VERSION_CHECK=on

WORKDIR /app

# Instalar dependencias del sistema
RUN apt-get update \
    && apt-get install -y --no-install-recommends \
    build-essential \
    default-libmysqlclient-dev \
    && apt-get clean \
    && rm -rf /var/lib/apt/lists/*

# Instalar dependencias
COPY requirements.txt .
RUN pip install -r requirements.txt

# Copiar el proyecto
COPY . .

# Exponer puerto
EXPOSE 8000

# Comando para iniciar el servidor
CMD ["gunicorn", "--bind", "0.0.0.0:8000", "mi_proyecto.wsgi:application"]

2. Crear docker-compose.yml

version: '3'

services:
  web:
    build: .
    restart: always
    volumes:
      - .:/app
      - static_volume:/app/staticfiles
      - media_volume:/app/media
    depends_on:
      - db
    env_file:
      - .env
    networks:
      - app_network
      
  db:
    image: mysql:8
    restart: always
    volumes:
      - mysql_data:/var/lib/mysql
    environment:
      MYSQL_DATABASE: ${DB_NAME}
      MYSQL_USER: ${DB_USER}
      MYSQL_PASSWORD: ${DB_PASSWORD}
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
    networks:
      - app_network
      
  nginx:
    image: nginx:latest
    restart: always
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d
      - static_volume:/app/staticfiles
      - media_volume:/app/media
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      - web
    networks:
      - app_network

networks:
  app_network:

volumes:
  mysql_data:
  static_volume:
  media_volume:

3. Crear configuración Nginx para Docker

Crea un directorio nginx/conf.d y un archivo nginx/conf.d/app.conf:

server {
    listen 80;
    server_name localhost;
    
    location /static/ {
        alias /app/staticfiles/;
    }
    
    location /media/ {
        alias /app/media/;
    }
    
    location / {
        proxy_pass http://web:8000;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Host $host;
        proxy_redirect off;
    }
}

4. Ejecutar con Docker Compose

docker-compose up -d

# Aplicar migraciones en el contenedor
docker-compose exec web python manage.py migrate

# Recoger archivos estáticos
docker-compose exec web python manage.py collectstatic --noinput
Ventaja de Docker: Con esta configuración, puedes desplegar tu aplicación en cualquier entorno que soporte Docker, como AWS ECS, Google Cloud Run o Kubernetes.

Paso 6.5: Tareas Post-Despliegue

Después de desplegar tu API, es importante realizar estas tareas adicionales:

Monitoreo

Configura herramientas de monitoreo para seguir el rendimiento y la disponibilidad:

  • Sentry para monitoreo de errores
  • Prometheus + Grafana para métricas
  • ELK Stack para análisis de logs
Seguridad

Implementa medidas de seguridad adicionales:

  • Configurar un firewall (UFW)
  • Implementar rate limiting
  • Configurar CORS adecuadamente
  • Actualización automática de paquetes
Respaldos

Configurar respaldos automáticos para la base de datos:

  • Respaldos diarios con rotación
  • Almacenamiento externo (S3, Google Cloud Storage)
  • Pruebas periódicas de restauración
CI/CD

Implementa integración y despliegue continuos:

  • GitHub Actions o GitLab CI
  • Pruebas automáticas
  • Despliegue automático después de pruebas exitosas
